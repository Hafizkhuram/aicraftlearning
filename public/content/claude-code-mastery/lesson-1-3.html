Claude Code: Setup, Workflow & Mastery Getting Claude Code onto your computer

Your first useful thing

A messy folder. One instruction in plain English. Ninety seconds of watching. By the end of this lesson, Claude Code will have done something genuinely useful on your machine — and it'll feel like a small trick.

The install is done. You've said hello. Now for the bit that turns Claude Code from "interesting new app" into "wait, where has this been all my life."

We're going to build a small practice folder full of the sort of mess that accumulates on every laptop — receipts, screenshots, loose PDFs, random images — and ask Claude Code to tidy it up. No risk to your real files. Just a safe playground for seeing what this thing can actually do.

!
The common misconception

"I should point Claude Code at my actual Downloads folder and watch it fix years of chaos." Tempting, but wrong — at least for now. Real folders often have files you care about. Lesson 1.4 covers the safety rules for working with real files; until then, we use a practice folder. This isn't a limitation. It's just sensible.

Build the practice folder (2 minutes)

Before Claude Code can tidy anything, we need something to tidy. We're going to create a folder called ClaudeCode-Practice on your desktop and drop some sample files into it. Think of it as the course's sandpit.

Create the folder

On your desktop, right-click an empty space → New Folder. Name it ClaudeCode-Practice. Exact name matters less than the fact that it's somewhere you'll find easily.

Fill it with "mess"

Drag in a handful of files from your real Downloads folder — 10 to 20 of varied types is perfect. If your Downloads is tidy (or empty), create some placeholder files instead: right-click in the folder, New → Text Document a few times, then rename each one something realistic like invoice-june.pdf, logo-draft.png, meeting-notes.docx, screenshot-2026-01-04.png.

The files don't need to be real PDFs or images — the names and extensions are what matter for this demo.

Open the folder in VS Code

In VS Code's menu bar, click File → Open Folder (on Mac) or File → Open Folder… (on Windows/Linux). Navigate to your desktop, select ClaudeCode-Practice, and click Open.

VS Code now shows your folder's contents in the sidebar on the left. This is the bit that matters: Claude Code can now see those files, and nothing else on your machine.

Why this matters: Claude Code's awareness is scoped to the folder you opened. It can read, move, and rename files inside ClaudeCode-Practice, but it has no access to anything outside it — not your Documents, not your real Downloads, not your work files. This is the single most important safety property of Claude Code, and it's automatic.

Have a look at the mess

Your folder probably looks something like this — a bit of everything, no structure:

📁 ClaudeCode-Practice/ invoice-june.pdf logo-draft.png Screenshot 2026-01-04 at 10.23.png meeting-notes.docx holiday-photo.jpg receipt_amazon.pdf proposal-v2-FINAL.docx proposal-v2-FINAL-final.docx logo-draft-v3.png invoice-july.pdf Screenshot 2026-01-18.png contract-draft.pdf

If you squint, you can see there's a logic here — some PDFs, some images, some documents, some screenshots. But nothing's sorted. If you had five years of this, you'd never find anything.

Here's where Claude Code earns its keep.

The one instruction that does the work

Click the spark icon in VS Code to open the Claude Code panel. In the prompt box at the bottom, type this exactly:

Claude Code · ClaudeCode-Practice
> Please organise this folder. Create subfolders by file type (PDFs, Images, Documents), move each file into the right subfolder, and leave a short summary.md at the top level explaining what you did.

Press Enter. Then stop and actually watch what happens.

Claude Code will do roughly the following — and it'll narrate each step as it goes:

  1. Read the folder and list every file inside it
  2. Group the files into logical categories based on extension
  3. Ask your permission to create new subfolders (you click Accept each time)
  4. Move each file into the right subfolder, one by one
  5. Write a short markdown file summarising what it did

The whole thing takes under two minutes on a folder of this size.

What you'll see on screen

Roughly, your conversation will look like this:

Claude Code · conversation
> Please organise this folder. Create subfolders by file type… I'll read the folder first to see what's in it. Found 12 files: 4 PDFs, 4 images, 2 documents, 2 screenshots. Here's my plan: • Create subfolders: PDFs, Images, Documents, Screenshots • Move files into their respective subfolders • Write summary.md at the top level Shall I proceed? > yes ⎯ Create folder: PDFs [Accept] [Reject] [Edit] [Accept] ✓ Created PDFs/ ✓ Created Images/ ✓ Created Documents/ ✓ Created Screenshots/ ✓ Moved 12 files ✓ Wrote summary.md Done. Here's what's in summary.md: ── This folder was organised on 22 April 2026. ── • 4 PDFs moved to /PDFs (invoices and contract) • 4 images moved to /Images (logos and photo) • 2 documents moved to /Documents (notes and proposal) • 2 screenshots moved to /Screenshots Nothing was deleted.

A few things worth noticing in that exchange.

First, Claude Code asked before making changes. That "Create folder" step didn't happen silently — you had to click Accept. This is the default safety setting, and it's doing exactly what it should. You can turn that off later once you trust a workflow, but for your first runs, every click is a reminder that you're still in charge.

Second, notice how it narrated its reasoning: "Found 12 files. Here's my plan." That's Claude Code's plan mode — it thinks out loud before acting. You can stop it and redirect it at any point.

Third, that summary.md file at the end is gold. Every time Claude Code does something meaningful, you can ask it to leave a written record. A month from now, you'll look at this folder and remember exactly what happened.

The bit that's supposed to surprise you

Have a look back at the instruction you typed:

Your prompt
Please organise this folder. Create subfolders by file type (PDFs, Images, Documents), move each file into the right subfolder, and leave a short summary.md at the top level explaining what you did.

Look closely. You didn't write any code. You didn't specify a scripting language. You didn't define a loop. You didn't handle edge cases. You described an outcome — "organise this folder, by type, leave a summary" — and Claude Code worked out every step in between.

This is the whole shift the course is about. Not "learn to write better code." Not "learn to use a more complicated tool." Just learn to describe what you want clearly, and let the AI handle the how.

The same pattern works for processing 50 PDF invoices. It works for cleaning 500 rows of customer data. It works for drafting 20 newsletters. The shape of the instruction is always the same: inputs, work, outputs, described in plain English.

The one thing to remember

You describe the outcome. Claude Code handles the steps. That's the entire shift.

The skill the rest of this course teaches isn't coding or command-line wizardry. It's describing what you want clearly enough that a capable assistant can act on it.

If it didn't quite work

A few small things can wobble on a first run. None are serious.

Claude Code asked too many permission questions. Normal on your first workflow. Click Accept each time and carry on. Later you'll learn to use auto-accept mode for workflows you trust — but today, every click is good practice.

The subfolders it created are named slightly differently from mine. Also normal. Maybe it used Docs instead of Documents, or split screenshots into their own folder when I didn't ask. That's Claude Code making a sensible judgement call. If you don't like it, just tell it: "Rename Docs to Documents" and it'll fix it.

It sorted things in a way I didn't want. Tell it: "Move the invoices into a separate Invoices folder instead of lumping them with other PDFs." This is iteration, and we cover it properly in Lesson 2.4.

It did nothing and said "I need more context." You probably opened VS Code on the wrong folder. Check the sidebar — the folder name at the top of the sidebar should read CLAUDECODE-PRACTICE. If it reads anything else, go to File → Open Folder and open the right one.
