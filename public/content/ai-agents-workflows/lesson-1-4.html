Getting n8n running in 15 minutes

The hardest part of building an agent isn't the agent — it's getting the tools set up. Let's remove that excuse right now.

This is the only setup-heavy lesson in the course. Fifteen minutes of signing up and clicking through settings, and you'll have the exact stack you need for Modules 2, 3, and 4. No servers. No Docker. No command line. Just a browser.

Module 1 has been about thinking. Module 2 onwards is about building. This lesson is the bridge.

!
The common misconception

"I need to learn DevOps, Docker, or servers before I can touch this properly." You don't. n8n offers a two-minute Cloud signup with no credit card, more than enough to finish every build in this course. You can decide where to host it for the long term after you've got systems actually saving you time. Not before.

Two paths — and which one to pick

There are genuinely two ways to run n8n. Both work. They're just different trade-offs between convenience now and cost later.

n8n Cloud (recommended for this course)

My recommendation: start with the Cloud trial. Signup takes two minutes, no credit card. By the end of the trial, you'll have real systems running and a much better idea of whether a $24/mo subscription is worth it, or whether you want to move to managed self-hosting to drop the cost to ~$3/mo. Both are fine outcomes. Neither decision has to be made today.

If you already have n8n set up from another course or past experiment, skip to Section 3 and just confirm your credentials are in place. No need to create a second workspace.

Sign up and sign in

Six steps. None of them take more than a minute. Work through them in order with a browser tab open to n8n.io.

Open the n8n website and click "Get started"

Go to n8n.io. The "Get started for free" button is in the top-right. That takes you to the signup page. Choose n8n Cloud (the hosted option), not the self-hosted download.

Screenshot · n8n.io home
Screenshot to add: n8n.io home page with the "Get started for free" button highlighted.

Create your account

Email and password, or sign in with Google. Pick whichever you'll actually remember. Your workspace URL will be yourname.app.n8n.cloud. This is your private n8n instance — no one else has access.

No credit card is required for the Starter/Pro trial. You'll only be asked for payment if you upgrade after the 14 days are up.

Land on your empty workspace

Once you're in, you'll see three main tabs in the left sidebar: Workflows, Credentials, and Executions. That's the entire app. Everything you'll ever do in n8n lives in one of those three places.

Screenshot · Empty workspace
Screenshot to add: the empty workspace view with Workflows / Credentials / Executions tabs visible in the sidebar.

Know what each tab does

Workflows is where your actual automations live — the visual builder, the nodes, the wiring. You'll spend most of your time here.

Credentials is where you store the connections to outside services — Claude, Gmail, Google Sheets, Slack. Set them up once, use them everywhere.

Executions is the history of every workflow run. When something breaks, this is where you look first. Module 2 has a whole lesson on it.

Complete the onboarding prompt

n8n will show a short onboarding flow — "what do you want to do?", "how experienced are you?". Answer honestly. The "non-technical, new to n8n" path surfaces AI-native templates and hides advanced settings you don't need yet.

Keep this tab open

You're in. That's it for signup. Leave the tab open — we're about to add your first credential, and then a tiny test workflow.

Add your first credential — Claude (or OpenAI)

A credential in n8n is a saved connection to an outside service. Once you've added one, any workflow in your workspace can use it. You added an API account in Lesson 1.3; now we're plugging it in.

Get your API key from the LLM provider

Open a new tab. Go to console.anthropic.com (or platform.openai.com for OpenAI). Find API Keys in the sidebar. Click Create Key. Copy the value — it starts with sk-ant- for Claude, or sk- for OpenAI.

One-time warning: you'll only see the full key once. If you lose it, you have to create another. Paste it straight into n8n in Step 3 below — don't leave it sitting in Notes or a chat message.

In n8n, go to Credentials → New

Back in your n8n tab. Left sidebar → Credentials → Add Credential. Search for Anthropic (or OpenAI) and click it.

Screenshot · Credentials panel
Screenshot to add: the Credentials panel with the search for "Anthropic" showing the right result.

Paste the key and save

Paste the API key into the field. Give the credential a name you'll recognise later — something like My Claude API. Click Save. n8n will test the connection; if the key is valid, you'll see a green tick.

If you see a red error instead, the most common cause is a copy-paste mistake — a leading space, or a truncated key. Go back to the provider console, regenerate the key, and try again.

Add one business-tool credential — Gmail

Pick the business tool you'll use most. For the Module 2 build, that's Gmail. If you're a Google Workspace user, this takes two clicks. If you're on Outlook or something else, the process is similar — just find your provider in the credential list.

Credentials → Add Credential → Gmail OAuth2

Back to the Credentials tab. Add Credential. Search for Gmail. Pick Gmail OAuth2 API. You don't need to configure anything in Google Cloud — n8n's hosted version handles that for you.

Click "Sign in with Google"

A standard Google OAuth popup appears. Pick the Gmail account you want to connect, review the permissions (n8n asks to read and send mail — this is expected), and approve.

Heads up: if you connect a personal Gmail account, Google will mark n8n as "unverified" in the popup. That's a standard warning for any app you've set up yourself. For a Workspace account you control, click through. For a shared or work inbox, check with your IT lead first.

Save and confirm the connection

Back in n8n, the credential saves automatically after OAuth. You should see a green tick. Two credentials in place — your LLM and your first business tool. That's enough to build with.

Your hello-world workflow

One small workflow to confirm everything works. Four nodes, two minutes to build.

Create a new workflow

Workflows tab → Add Workflow. You'll see an empty canvas with a big "+" button in the middle. Click it to add your first node.

Add a Manual Trigger

Search for Manual Trigger and add it. This lets you run the workflow on demand by clicking a button — useful for testing. Later builds will use other triggers (schedule, email received, webhook), but Manual is perfect for a first run.

Add a Claude (or OpenAI) node

Click the "+" next to your Manual Trigger. Search for Anthropic (or OpenAI). Pick the "Message a Model" action.

In the node's settings: pick the credential you just saved. Under Model, choose the recommended default (Sonnet for Claude, the default GPT for OpenAI). In the Message field, type: Say hello and tell me today's date in one sentence.

Screenshot · LLM node config
Screenshot to add: the Anthropic / OpenAI node config panel with credential selected and the test message in the Message field.

Run it

Click Test step on the Claude node. Within a second or two, you'll see the response appear in the right-hand panel — something like "Hello! Today's date is…". That output is what any downstream node would receive.

Congratulations. You've just sent a real query to a frontier LLM from inside your own workflow. Everything else in this course is a bigger, more useful version of exactly that.

The one thing to remember

You don't need to understand how n8n works under the hood. You need two credentials saved and one workflow running. That's today's bar.

Everything else — the builds, the patterns, the guardrails — stacks on top of this. Get it working now and the rest of the course flows.

If something went wrong

Most first-time setup issues fall into three buckets. If you hit one, here's the fix.

Credential saves but the test fails. Almost always a billing issue — the API account needs credit added before it can answer requests. Pop back to the provider console, add $5 if you haven't already, and retest. The credential is fine; the account just wasn't funded.

OAuth popup keeps failing. Check that third-party cookies aren't blocked in your browser. Incognito mode is the quickest fix. Alternatively try a different browser for the OAuth step — after it saves, you can go back to your usual browser.

The Claude/OpenAI node doesn't appear in the search. Your workspace might be on an older template. Close the canvas, open a fresh workflow, and search again. If that still fails, the node you want is sometimes listed as "AI" → "Anthropic Chat Model" rather than a top-level result.
