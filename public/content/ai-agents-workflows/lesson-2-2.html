Building AI Agents & Workflows Build your first workflow

Build #1 — email triage workflow

Your inbox is a firehose. By the end of this lesson, it's a sorting system — with drafts written for the routine stuff, waiting in your drafts folder for a one-second review before you hit send.

This is the first real build. Bigger than the hello-world from Lesson 1.4, but still small enough to finish in one sitting. Everything you learned in Module 1 and in Lesson 2.1 comes together here: a fixed path (workflow, not agent), a boring-but-frequent task (high ROI potential), a small stack (n8n + Claude + Gmail), and a human-in-the-loop at the end so nothing goes out without you seeing it.

!
The common misconception

"Build #1 should auto-reply to customers — that's the real time-saver." It isn't, and it's dangerous. Auto-sent AI replies going to customers is exactly how the high-profile failures in the news happened. This lesson builds the drafts-only version — AI writes, you approve. Cuts 80% of the reply time, keeps 100% of the judgement. That's the pattern for every customer-facing build in this course.

What you're building

Build #1 spec

Inbox triage with drafts-only replies

  • When: a new email arrives in your Gmail inbox.
  • Classify it with Claude as one of four types — urgent, routine, spam, or request.
  • Label the email in Gmail based on that classification, so your inbox self-sorts.
  • If the email is routine, draft a polite reply with Claude and save it to Gmail drafts — never sent automatically.
  • You review the draft with one click and hit send when ready. Or don't. Or edit first. Your call, every time.

One trigger. Two actions (label + draft). The IF node in the middle is the key — it's what stops the workflow trying to draft replies to spam. We'll come back to why that pattern matters.

Step 1 — Set up Gmail labels first

Before building the workflow, create four labels in Gmail. This takes ninety seconds and makes the rest of the build vastly easier.

Open Gmail and create four labels

In Gmail's left sidebar, scroll to Labels → + Create new label. Create these four exactly (lowercase, with the ai-triage/ prefix so they group together):

  • ai-triage/urgent
  • ai-triage/routine
  • ai-triage/spam-like
  • ai-triage/request

The prefix turns these into a nested group you can collapse. Your inbox stays tidy and you can scan by category at a glance.

Why lowercase and hyphenated? n8n passes the classification label from Claude straight to Gmail. Matching the case exactly means no "label not found" errors later. Save yourself the debug and keep it simple.

Step 2 — The trigger

Back to n8n. Create a new workflow (call it Email triage) and add the Gmail Trigger.

Add a Gmail Trigger node

Click the "+" on the canvas. Search for Gmail Trigger (not the regular Gmail node). Pick the On message received event.

Screenshot · Gmail Trigger config
Screenshot to add: the Gmail Trigger node with credentials selected, poll interval set, and filters visible.

Configure it like this:

Credential
The Gmail credential you saved in Lesson 1.4.
Poll Times
Every minute (default). Pings Gmail once a minute — delay is ~60s.
Simplify
On. Returns clean email fields — subject, from, body snippet.
Include Spam and Trash
Off. Don't waste classifications on known spam.
Read Status
Unread only. We only care about new, unopened mail.

Click Test step. n8n will fetch the most recent unread email and show it in the right-hand panel. That's your test data for the rest of the build.

Heads up: n8n polls every minute — it's not a real-time webhook. An email you send right now might take up to 60 seconds to show up. That's fine for triage; it's not fine for anything that feels urgent to the sender. Worth remembering.

Step 3 — The classifier

This is the heart of the workflow. A single Claude call that reads the email and returns exactly one word: urgent, routine, spam, or request.

Add an Anthropic "Message a Model" node

Click "+" after the Gmail Trigger. Search for Anthropic. Pick the Message a Model action (same node type as your hello-world in Lesson 1.4).

Settings:

Credential
Your Claude credential from Lesson 1.4.
Model
claude-sonnet-4-6 — the sweet spot on cost and quality.
Message
Paste the prompt below (expressions enabled so it can read the email).

Paste this prompt into the Message field. The three {{ green expressions }} pull data from the Gmail Trigger node above.

Classifier prompt Paste into Message field
# Task You are classifying an incoming email into exactly one of four categories: - urgent: time-sensitive; needs a human reply today (e.g. client crisis, broken service, angry complaint) - routine: standard back-and-forth; a templated or boilerplate reply is appropriate (e.g. availability question, delivery update, meeting confirmation) - spam: marketing, cold sales, clearly automated (e.g. promotional emails, newsletter blasts, bulk outreach) - request: someone is asking you to do something specific that needs your judgement (e.g. proposal request, intro favour, new project brief) # Rules - Reply with exactly one word from that list. - Lowercase. No punctuation. No explanation. - If genuinely unsure between two, pick the one with higher human-attention cost (urgent over routine, request over routine). # Email to classify From: {{ $json.From }} Subject: {{ $json.Subject }} Body: {{ $json.snippet }}

Click Test step. You'll see one word come back in the output panel. That one word is what the rest of the workflow routes on.

Why one word, not a paragraph? Short, constrained outputs are dramatically more reliable. Asking for "a paragraph explaining which category and why" invites the model to drift, hedge, and occasionally return two categories. Asking for one word almost never misfires. Constrain the output; trust the process.

Step 4 — The IF branch

Now the split. We only want to draft replies for routine emails. For everything else, we skip the drafting step and go straight to labelling.

Add an IF node

Click "+" after the Claude classifier. Search for IF. Add the IF node.

Configure one condition: String equals.

Value 1
The Claude output — drag in the expression {{ $json.content[0].text }}.
Operation
is equal to
Value 2
routine

The IF node now has two output branches: true (email was classified as routine) and false (it wasn't). We'll wire different things to each.

This is the human-in-the-loop checkpoint you've been promised. Spam doesn't get a draft. Urgent doesn't get a draft. Requests don't get a draft — because you should be answering requests yourself. Only the boring, templatable middle of the distribution gets automated, which is exactly where the ROI lives.

Step 5 — The drafter (routine branch only)

On the true branch, add a second Claude node. This one writes the actual reply.

Add another Anthropic "Message a Model" node

Connected to the IF node's true output. Same credential, same model. Different prompt — this one writes the reply.

Drafting prompt Paste into Message field
# Task You are drafting a reply to a routine email. The reply will be saved as a draft for a human to review before sending. # Rules - Keep it warm, brief, and professional. - British English spelling (organise, colour, favour, etc). - No more than 3 short paragraphs. - Never invent facts — if you don't know a specific date, number, or name, leave a placeholder in square brackets like [date] or [your details] for the human to fill in. - No sign-off — the human will add that. - No subject line — this is a reply; Gmail keeps the thread subject. # Original email From: {{ $('Gmail Trigger').item.json.From }} Subject: {{ $('Gmail Trigger').item.json.Subject }} Body: {{ $('Gmail Trigger').item.json.snippet }} # Draft the reply body below

Notice the $('Gmail Trigger') syntax — it reaches back past the classifier to grab the original email. You need this because the previous node (the classifier) only returned a single word; the email content itself lives on the Gmail Trigger node.

Step 7 — The label action (runs on both branches)

Finally, apply the label. This runs for every email regardless of classification.

Add a Gmail → Message → Add Label node

Connect it so it runs on both the IF node's true branch (after the draft is created) and the false branch. In n8n this is just a matter of dragging an arrow from both branches into the same node.

Resource
Message
Operation
Add Label
Message ID
Expression: {{ $('Gmail Trigger').item.json.id }}
Label Name
Expression: ai-triage/{{ $('Message a Model').item.json.content[0].text }} — pulls the classifier output and stitches it onto the prefix.

The last expression is the clever bit. Claude returns routine (or urgent, or spam, etc.). We concatenate it with ai-triage/ so the full label becomes ai-triage/routine — which matches the labels you created in Step 1.

Step 8 — Switch it on and watch it work

At the top of the n8n canvas there's a toggle — currently it says Inactive. Toggle it to Active. Your workflow is now running.

Save, activate, and let it process 10 emails

Save the workflow. Toggle it active. Go about your day.

Over the next few hours, as emails arrive, you'll see:

  • Each email gets labelled within about a minute of arriving
  • Routine emails get a draft waiting in your Gmail drafts folder
  • Spam, urgent, and request emails get labelled but no draft
  • The Executions tab in n8n shows every run, success or failure

Click Executions after an hour. You should see 5-15 entries. Click any of them to see exactly what happened at each node — what the classifier said, what the draft looked like, which branch it took. That view is the single most useful thing in n8n for debugging.

The one thing to remember

Never let the first version send anything. Drafts first, sent later — always.

The IF branch and the "Create Draft" action together are the human-in-the-loop. Every future build in this course follows the same pattern — AI decides what could be done; you decide what actually gets done.

What to watch for in the first week

Your workflow is running. Three things to keep an eye on.

Classification accuracy. Scan the labels for a few days. Any obviously miscategorised emails? If Claude keeps calling customer questions "spam," the prompt needs tightening — add a concrete example of a customer question to the prompt so Claude has a clearer reference.

Draft quality. Open a few of the routine drafts. If they're too formal, too stiff, or missing your voice, edit the drafting prompt — add one or two lines like "write in a warm, friendly tone, similar to how a small business owner would reply." Prompts are text you iterate on, not code you set and forget.

Cost. Check your Anthropic console after a week. For 500 emails/month you should be seeing maybe £2-5 in usage. If it's much higher, either you're getting far more email than you thought (fair), or a subtle bug is making duplicate calls (worth checking the Executions log).

Want more classification categories? Add them to the classifier prompt, and create a matching label in Gmail. newsletter, internal, personal — whatever your inbox actually contains. The workflow scales to however many categories you want because the labelling step uses string concatenation, not a hardcoded list.
