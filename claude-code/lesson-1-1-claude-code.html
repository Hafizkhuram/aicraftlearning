Claude Code: Setup, Workflow & Mastery Getting Claude Code onto your computer

What Claude Code actually is

Claude Code has "code" in the name and lives in something called a terminal. That scared me off for months. Here's why I was wrong — and why this might be the most useful AI tool you're not using yet.

Let's get the awkward thing out of the way. The tool we're about to spend 14 lessons on is called Claude Code. The word "code" is right there in the name. It has a command-line interface. Screenshots of it make it look like something your nephew uses to hack the school network.

If any of that made you feel slightly out of your depth, you're in the right course. Because that name, and that first impression, is the biggest reason most business owners never touch the tool that would save them the most time. By the end of this lesson, the fear should be gone. By the end of this course, you'll be using Claude Code every week without thinking twice.

!
The common misconception

"Claude Code is a tool for developers." It sounds obvious — the product literally has "Code" in the name, it runs in a terminal, and every tutorial you've seen probably involves someone building an app. But inside Anthropic, across agencies, and among solo business owners, Claude Code is now mostly being used for work that has nothing to do with software.

The real difference from the Claude you already know

You've probably used Claude (or ChatGPT) in your browser. You open a tab, type a question, and it replies. That's the chat Claude — useful, but strictly a conversation.

Claude Code is different in one specific way: it works on your actual files, on your actual computer. Not a paste-in. Not an upload. Your real folders, spreadsheets, PDFs, and documents — the ones already sitting on your desktop.

That one difference is the entire point of this course. Here's the comparison in plain terms:

The chat Claude (in your browser)
Claude Code (on your computer)
Can answer questions and draft text you paste in.
Can act on files already sitting in your folders.
Processes one thing at a time.
Processes dozens of files in a single run.
Forgets you the moment you close the tab.
Can remember your project, preferences, and voice.
Great for conversations and one-off drafts.
Great for repeatable, real-world workflows.

If the chat Claude is a conversation, Claude Code is a colleague. One you can ask to clean up 200 invoices while you make a cup of tea.

An analogy that sticks

Picture this

The chat Claude is a brilliant consultant you hire by the hour. You describe your problem, they give you excellent advice, you thank them, they leave. Nothing actually gets done — but you have good advice to act on.

Claude Code is a brilliant assistant with a key to your office. You describe what needs doing, they go do it on your desk, and when you come back the files are sorted, the report is drafted, and a summary is waiting for you on a sticky note.

Both are valuable. They're just different jobs. The chat Claude helps you think. Claude Code helps you finish. Most business owners have only met the first one, which is why they feel stuck at "AI is interesting but it doesn't really save me time."

Claude Code is where the time-saving actually lives.

What it looks like in practice

Here's a glimpse of what a Claude Code session looks like — not because you need to memorise it, but so the phrase "terminal" stops feeling like it belongs in a Matrix film. It's just a text window where you tell Claude what to do, and it replies with what it's doing.

claude-code · ~/Invoices-October
> Read every PDF in this folder. For each one, pull out the supplier name, invoice number, amount, and due date. Save the results as a single spreadsheet called october-invoices.xlsx. Claude Code is reading 47 files… ✓ Extracted data from 47 invoices ✓ Normalised currency and date formats ✓ Saved to october-invoices.xlsx ✓ Flagged 2 invoices with unclear due dates for review Done in 1 min 14 sec.

That's a real task. That's roughly two hours of human work, done in under ninety seconds, without writing a single line of code. The only thing you did was describe what you wanted in ordinary English.

This is the shape of almost every workflow in this course. You describe the outcome. Claude Code handles the steps. No scripting, no automation platform, no developer on speed dial.

Who's actually using it like this

If you're still wondering whether non-coders really use Claude Code this way, the honest answer is: yes, and in quietly large numbers. Examples from recent reporting and Anthropic's own teams:

  • A marketing operator built a personalised-content system in about three hours. It costs around 15 cents a week to run.
  • A recruiter paste-drops interview transcripts and gets back a structured analysis and a ready-to-use Notion card, with suggested next steps.
  • An agency owner processes 54 PDF invoices at once, in parallel, and lands them in a clean spreadsheet before the kettle boils.
  • A consultant feeds hundreds of ad headlines from a CSV into Claude Code and gets back a scored, improved version of each one.

None of those people are developers. They're the same kind of person this course is built for: they've got real work, limited time, and no interest in learning to code. They just learned how to describe what they want clearly — and that's the skill this course teaches.

The one thing to remember

Claude Code isn't a coding tool. It's an AI that works on your files, not just your chat window.

The name will scare off most business owners. The ones who get past it are the ones building real automation in their work this year.

What to ignore for now

Before you start Googling Claude Code, a small warning. Most of what's written about it online is aimed at developers — people building apps, debugging software, writing scripts. That content isn't wrong, it's just not for you.

You will see tutorials about git, npm, testing frameworks, and something called "agentic engineering." You can safely ignore all of it. We won't touch any of that in this course. If a YouTube video mentions "vibe coding" in the first 30 seconds, close the tab.

What you will learn in the next 13 lessons is a focused, business-first path: install the tool, set up one good project, run real workflows on real files, and package the ones you repeat. That's the entire scope. Nothing more, nothing that smells like software development.

One final note on fear. The terminal looks like it was designed by someone who really enjoys the 1970s. It was. But we're going to mostly avoid it by using the VS Code extension — a familiar window with a chat panel on the side. Next lesson, that install is the whole job.
