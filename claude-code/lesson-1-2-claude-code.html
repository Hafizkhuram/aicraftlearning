Claude Code: Setup, Workflow & Mastery Getting Claude Code onto your computer

Installing Claude Code the easy way

There are two ways to install Claude Code. One looks like something out of The Matrix. The other looks like a normal app. Guess which one I'm recommending.

This is the lesson where the word "terminal" stops being scary. You're going to have Claude Code running on your computer by the end of the next ten minutes, and you're going to do it using a normal-looking window with a normal-looking sign-in button. No obscure commands. No dependencies to install. No "wait, do I have Homebrew?"

Open your laptop, make a cup of tea, and let's get this done.

!
The common misconception

"You need to be comfortable with the terminal to install Claude Code." That was half-true a year ago. It's not true now. The friendliest way to install Claude Code is through a free, visual application called VS Code — and it does all the terminal stuff for you, behind the scenes.

Before you start — the three things you need

Before we install anything, let's make sure you've got the three things the install needs. Don't worry if you haven't — none of them take more than a few minutes to sort out.

  • A laptop or desktop running macOS, Windows, or Linux. A work machine is fine, but if your IT team locks down installs you may need their help.
  • A Claude account with a Pro or Max subscription. The free plan doesn't include Claude Code. Pro is $20/month and is the right starting point — you can upgrade to Max later if you end up using it daily.
  • About 20 minutes of uninterrupted time. Longer if your internet is slow or you hit a wobble, which I'll help you through.
A word on the subscription: your research spec might tell you to jump straight to Claude Max ($100-200/month). Don't. Start on Pro. Use Claude Code for two weeks. If you're genuinely using it every day and bumping into rate limits, then upgrade to Max. Buying the expensive plan first is the classic way to waste money on a tool you never build the habit of using.

The two paths — and why we're taking the easy one

You can install Claude Code in two ways. It's worth knowing both exist, even though we're only using one.

The hard way — pure terminal
The easy way — VS Code extension
You install a program called claude using a command called npm, then run it in a black text window called a terminal.
You install a free app called VS Code, then add a Claude Code extension to it. Claude Code runs in a normal-looking chat panel.
Requires you to have something called Node.js installed, plus some comfort with command-line tools.
Requires nothing but VS Code itself. The extension bundles everything else for you.
Great for developers who already live in the terminal.
Great for everyone else. Same power, far less friction.

If you hear any developer friends telling you that "proper" Claude Code is the terminal version, you can ignore them. The two approaches talk to the exact same AI, share the same settings, and can even share the same conversation history. The only difference is which window you sit in. We're picking the window that doesn't look like a hacker film.

Installing VS Code (5 minutes)

VS Code is a free code editor from Microsoft. You don't need to know anything about code to use it — in our case, it's just the container that Claude Code lives inside. Think of it as the picture frame; Claude Code is the picture.

Download VS Code

Go to code.visualstudio.com in your browser. Click the big blue download button — it'll automatically offer the right version for your operating system (Mac, Windows, or Linux).

Install it like any other app

On Mac, drag it into your Applications folder. On Windows, run the installer and accept the defaults. Nothing fancy — it's a normal app install. It takes about a minute.

Open it once to let it settle

Launch VS Code. You'll see a Welcome tab, probably offering to install themes and colour options. Ignore all of it for now. The only thing that matters is that VS Code is open and working. Close the Welcome tab. Now you're ready.

One version check: Claude Code needs VS Code version 1.98.0 or later. If you've just downloaded it fresh, you're fine — it'll be the latest. If you've had VS Code sitting on your machine for a while, go to the Help menu (or Code menu on Mac) and click Check for Updates before continuing.

Adding the Claude Code extension (3 minutes)

Now the actual Claude Code install. You're adding it to VS Code the same way you'd add any extension — through a built-in marketplace.

Open the Extensions panel

On the left edge of VS Code there's a narrow strip of icons called the Activity Bar. Click the one that looks like four squares (it's the Extensions icon). Or press Cmd+Shift+X on Mac, Ctrl+Shift+X on Windows and Linux.

Search for "Claude Code"

A search box appears at the top of the panel. Type Claude Code into it. Several results will come up.

Install only the one published by Anthropic — you'll see the word "Anthropic" next to the name, and usually a verified tick. Other extensions are third-party and not what we want.

Click Install

Blue Install button. Click it. It takes about 20 seconds. When it's done, you'll see a new icon in the Activity Bar — a small "spark" shape. That's Claude Code.

Signing in and saying hello (2 minutes)

Almost there. Now you just need to tell Claude Code who you are, and confirm the whole thing works.

Open the Claude Code panel

Click the spark icon in the Activity Bar. A panel opens — either on the side or as a new tab. It'll ask you to sign in.

Sign in through your browser

Click Sign in. It'll open a browser tab asking you to authorise VS Code. Use the same Claude account that has your Pro or Max subscription. Once it's done, the browser will tell you to close the tab and return to VS Code.

Say hello

Back in VS Code, you'll see a prompt box at the bottom of the Claude Code panel. Type hello and press Enter. Claude will reply within a second or two.

That's it. You've just had your first exchange with Claude Code. Take a breath. The hard part is over.

Here's what that first exchange looks like, roughly:

Claude Code · VS Code panel
> hello Hi! I'm Claude Code. I can read and write files in your current workspace, run commands, and help with all sorts of tasks — from processing documents to drafting content. What would you like to start with?
The one thing to remember

Install the VS Code extension. It's the gentlest entry point, and it gives you everything Claude Code can do.

There is no "proper" or "advanced" version you're missing out on. The extension and the terminal version run the same AI, share the same settings, and can even share conversation history.

If something went wrong

Installs usually go smoothly. But a few specific things trip people up, and they're all easy to fix. If you hit a snag, try these in order.

I don't see the spark icon after installing

Two common causes. First, the extension sometimes needs VS Code to reload. Open the Command Palette (Cmd+Shift+P on Mac, Ctrl+Shift+P on Windows/Linux), type Developer: Reload Window, and press Enter.

Second, you may have installed a lookalike extension. Go back to the Extensions panel, uninstall anything called "Claude Code" that isn't published by Anthropic, then install the official one.

It keeps asking me to sign in every time I open VS Code

This usually happens when your Claude session hasn't saved properly. Sign out fully (click your profile in Claude Code, choose Logout), close VS Code, reopen it, and sign in again. That tends to sort it.

If it still happens, check that your Claude subscription is active in your account at claude.ai. An expired or cancelled subscription is the most common cause of being kicked back to the login screen.

Sign-in says my account doesn't have access

You're almost certainly on the Free plan. Claude Code requires Pro or Max. Go to claude.ai, click your profile, and upgrade to Pro. Then come back to VS Code and sign in again — your new subscription will be recognised immediately.

I typed "hello" and Claude never replied

Check your internet connection first — Claude Code needs an online connection to work. If that's fine, try starting a new conversation (there'll be a "New conversation" option in the panel). If the issue persists, reload VS Code as described above. In the rare case nothing works, the docs have a full troubleshooting page you can check.

My work laptop won't let me install VS Code

You've hit an IT policy, not a technical problem. Talk to your IT team and explain you need VS Code (a Microsoft-made open-source code editor) for a professional development course. Most IT teams will approve it, since it's one of the most widely-used business tools in the world.

If they say no, you have two options: use a personal device for this course, or ask them to install it for you in managed mode.

A quick word on Pro vs. Max

You'll see a lot of noise online about which Claude subscription to use. Here's the honest version.

Claude Pro
Claude Max
$20/month Generous daily usage for most people running Claude Code on their own projects. More than enough for this course and the workflows we'll build.
$100-$200/month 5x-20x higher usage limits. Only makes sense once you're running Claude Code every day and hitting Pro's limits regularly.
Start here. Every learner I know who began on Max spent the first month barely using it.
Upgrade later, only if you genuinely run out of room on Pro. Your future self will thank you for not overspending early.

The exact pricing sometimes changes, but the principle doesn't: start on the cheaper plan, upgrade when you have real evidence you need to. That's true for any subscription, and it's especially true for a tool you're just beginning to build a habit around.
